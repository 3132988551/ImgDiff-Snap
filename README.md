# ImgDiff Snap

把差异“拉出来看”的小工具：拖两张同尺寸图片进来，分割对比像拉帘子，一键导出原尺寸合成图；切到热力图，哪里变了红色一眼看穿，阈值可调，结果同样可导出。

> 不过眼睛会骗你；但是像素不会。  
> 不过“看着差不多”；但是热力图一开，变化立刻现形。  
> 不过怕导出不清楚；但是给你原图大小的 PNG。  
> 不过分割线只是用来看；但是导出时不会把它画进去。

## 功能亮点（人话版）
- 分割视图：像拉帘子一样左右拖动查看前后变化；“拖到哪，就按哪导出”，不绘制分割线。
- 热力图视图：按阈值把差异映射为红色透明度，越红变化越大；支持自定义阈值并显示变化占比。
- 一键导出：分割合成图、热力图均可导出原尺寸 PNG（文件名包含参数与时间戳）。
- 顺手小功能：交换左右、键盘微调（←/→，按住 Shift ×10）、内置示例、离线可用。

## 使用方式（3 步）
1. 拖入或选择两张同尺寸的 PNG/JPEG/WebP。
2. 在“分割”视图自由拖动分割线，或切到“热力图”视图调整阈值。
3. 点击右上角按钮：
   - 分割视图 → “下载分割图”（导出按当前比例合成的整图，不画分割线）。
   - 热力图视图 → “下载热力图”。

## 技术栈与实现
- React 18 + TypeScript + Vite（纯前端、HMR 友好）。
- Canvas 2D 渲染与导出，文件不经服务器。
- Web Worker 进行像素级计算，避免主线程卡顿（≥6MP 自动启用）。
- 离线与隐私：100% 本地运行，无任何网络请求；构建后为纯静态站点，可离线使用。

## 限制与小贴士
- 仅支持同尺寸图片；单图 ≤ 16MP，单边 ≤ 8192px。
- 透明像素（α < 255）不参与比较，结果更干净，不受背景干扰。
- 分割导出遵循“拖到哪就导哪”，像素对齐到整数，避免缝隙/重叠。

## 快速开始
- 环境：Node.js ≥ 18
- 安装：`npm i`
- 开发：`npm run dev`（Vite 本地预览）
- 构建：`npm run build` → 产出到 `dist/`
- 预览构建：`npm run preview`
- 检查：`npm run check`（ESLint + TypeScript + Build）
